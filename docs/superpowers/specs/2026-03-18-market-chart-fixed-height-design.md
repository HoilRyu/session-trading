# 시세 차트 데스크톱 고정 높이 레이아웃 설계서

## 개요

- 작성일: 2026-03-18
- 대상 경로: `frontend/src/features/market-chart/components/`
- 목적: 데스크톱 `시세 / 차트` 화면에서 마켓 목록이 길어져도 전체 페이지와 차트 영역 높이가 함께 늘어나지 않고, 마켓 목록 패널 내부에서만 스크롤되도록 레이아웃 규칙을 정의한다.

## 목표

- 데스크톱 `시세 / 차트` 화면 높이가 viewport 기준으로 고정돼야 한다.
- 오른쪽 마켓 목록 패널은 부모 높이 안에서만 표시돼야 한다.
- 마켓 목록이 길어져도 차트 영역과 페이지 전체 높이는 늘어나지 않아야 한다.
- 실제 스크롤은 마켓 목록 본문 영역에서만 발생해야 한다.

## 범위

- 데스크톱 `시세 / 차트` 레이아웃 높이 제약
- 오른쪽 마켓 목록 패널 높이 제약
- 리스트 본문 내부 스크롤 유지
- 현재 테스트를 고정 높이 규칙 기준으로 보강

## 비범위

- 모바일 레이아웃 수정
- 마켓 목록 API 구조 변경
- 차트 심볼 변경
- 마켓 선택 동작 변경
- UI 디자인 리뉴얼

## 설계 결정

### 1. 데스크톱 콘텐츠 그리드가 남은 화면 높이를 모두 차지한다

현재는 데스크톱 `시세 / 차트` 화면이 `상단 영역 + 콘텐츠 그리드` 구조이지만, 콘텐츠 그리드 자체가 viewport 높이에 고정되지 않아 마켓 목록 길이에 따라 전체 높이가 늘어날 수 있다.

이번 수정에서는 최상위 컨테이너가 화면 높이를 기준으로 고정되고, 상단 영역을 제외한 남은 높이를 콘텐츠 그리드가 모두 차지하도록 바꾼다.

즉 구조는 아래처럼 유지한다.

```text
데스크톱 시세 / 차트
├─ 상단 영역 (고정 높이)
└─ 콘텐츠 그리드 (남은 높이 전체)
   ├─ 왼쪽 메인 컬럼
   └─ 오른쪽 마켓 목록 패널
```

### 2. 왼쪽 메인 컬럼과 오른쪽 목록 패널 모두 `min-h-0`로 줄어들 수 있어야 한다

CSS grid/flex 조합에서는 자식이 `min-height: auto` 상태면 내부 콘텐츠 크기만큼 부모를 밀어 올릴 수 있다.

이번 수정에서는:

- 콘텐츠 그리드
- 왼쪽 메인 컬럼
- 오른쪽 목록 패널

모두가 `min-h-0` 상태에서 부모 높이 안에 갇히도록 만든다.

이 규칙이 있어야:

- TradingView 차트 영역이 남은 공간에 맞춰 유지되고
- 오른쪽 목록 패널도 부모 높이를 넘지 않으며
- 실제 넘치는 내용은 내부 스크롤로만 처리된다.

### 3. 마켓 목록 패널은 전체 패널이 아니라 본문 리스트만 스크롤된다

상단 탭과 헤더는 항상 고정된 상태로 보여야 한다.

따라서 마켓 목록 패널은 아래 구조를 유지한다.

```text
마켓 목록 패널
├─ quote 탭
├─ 컬럼 헤더
└─ 리스트 본문 (overflow-y-auto)
```

즉, 패널 전체가 스크롤되는 것이 아니라:

- `원화 / BTC / USDT` 탭은 고정
- `종목명 / 현재가 / 전일대비 / 거래대금` 헤더도 고정
- 리스트 본문만 스크롤

이 되어야 한다.

### 4. 차트 영역은 목록 길이와 무관하게 현재 레이아웃 높이를 유지한다

왼쪽 메인 컬럼은:

- 상단 정보 영역
- TradingView 차트 영역

으로 구성되어 있다.

이번 수정의 핵심은 오른쪽 목록이 길어져도 왼쪽 차트 영역이 같이 늘어나지 않도록 하는 것이다.

따라서 차트 영역은:

- 부모 높이를 기준으로 `flex-1`
- 최소 높이는 유지
- 목록 패널 길이와 독립

구조를 유지한다.

## 데이터 흐름 영향

이번 작업은 레이아웃 수정이므로 데이터 흐름 자체는 바꾸지 않는다.

기존 흐름은 그대로 유지한다.

```text
quote 탭 선택
→ useMarketList({ quote })
→ GET /api/v1/markets
→ 목록 데이터 렌더링
```

변경되는 것은 렌더링 높이 제약뿐이다.

## 테스트 기준

아래를 테스트로 고정한다.

- 데스크톱 `시세 / 차트` 최상위 컨테이너가 고정 높이 레이아웃 클래스를 가진다.
- 콘텐츠 그리드가 `min-h-0`과 남은 높이 점유 구조를 가진다.
- 오른쪽 마켓 목록 패널이 부모 높이를 채우는 클래스를 가진다.
- 리스트 본문은 여전히 `overflow-y-auto`를 유지한다.

## 검증 기준

아래 조건을 만족하면 구현 완료로 본다.

- 데스크톱 `시세 / 차트`에서 마켓 목록이 길어져도 페이지 전체 높이가 늘어나지 않는다.
- 차트 영역 높이가 마켓 목록 길이에 의해 같이 늘어나지 않는다.
- 마켓 목록은 패널 내부에서만 스크롤된다.
- `pnpm test:run`, `pnpm build`가 모두 통과한다.

## 가정

- 현재 데스크톱 `시세 / 차트`는 viewport 기준 높이 제약만 추가하면 원하는 동작을 만들 수 있다.
- 모바일 레이아웃은 이번 수정 범위에 포함되지 않는다.
