# 시세 차트 모바일 목록 레이아웃 설계서

## 개요

- 작성일: 2026-03-17
- 대상 경로: `frontend/src/pages/HomePage.tsx`
- 목적: 모바일 `시세 / 차트` 진입 화면에서 콘텐츠 영역을 `마켓 목록 영역`으로 분리해, 이후 목록/상세 라우트 구조로 확장 가능한 기본 골격을 만든다.

## 목표

- 모바일 `/market-chart` 진입 시 기존 공통 콘텐츠 블록 대신 `마켓 목록 영역`이 보이도록 한다.
- 다른 모바일 메뉴 화면은 기존 공통 콘텐츠 표현을 유지한다.
- 이후 `/market-chart/:market` 상세 라우트로 확장해도 현재 구조를 다시 뜯어고치지 않도록 경계를 분리한다.

## 범위

- 모바일 `시세 / 차트` 목록 화면의 콘텐츠 영역 교체
- `시세 / 차트` 모바일 목록 화면 전용 레이아웃 컴포넌트 분리
- 홈 레이아웃에서 현재 경로에 따라 모바일 콘텐츠 영역을 분기하는 구조 설계

## 비범위

- 모바일 `시세 / 차트` 상세 화면 구현
- 실제 마켓 아이템 리스트 렌더링
- 마켓 아이템 클릭 시 라우팅 연결
- 데스크톱 `시세 / 차트` 레이아웃 변경
- 실제 데이터 연동, 상태관리, 차트 라이브러리 연결

## 설계 결정

### 1. 모바일 목록/상세 구조 방향

모바일 `시세 / 차트`는 상태 전환이 아니라 라우트 분리 방식을 전제로 한다.

- 목록 화면: `/market-chart`
- 상세 화면: `/market-chart/:market`

이번 단계에서는 목록 화면만 레이아웃으로 표현한다. 다만 현재 구조가 이후 상세 라우트 추가를 방해하지 않도록, `시세 / 차트` 모바일 콘텐츠를 공통 `HomePage` 안에 직접 하드코딩하지 않고 작은 전용 컴포넌트로 분리한다.

### 2. 현재 단계의 모바일 목록 화면

모바일 `/market-chart` 진입 시에는 콘텐츠 영역에 아래 라벨만 보이게 한다.

- `마켓 목록 영역`

즉, 지금까지 공통으로 보이던 `콘텐츠 영역 - 시세 / 차트`를 모바일 `시세 / 차트`에 한해서만 교체한다.

### 3. 적용 범위 제한

이번 변경은 오직 모바일 `시세 / 차트`에만 적용한다.

- `/market-chart` 모바일: `마켓 목록 영역`
- 그 외 모바일 메뉴: 기존 `콘텐츠 영역 - 메뉴명`
- 데스크톱 `시세 / 차트`: 기존 전용 데스크톱 레이아웃 유지

이 제한은 현재 요청 범위가 `영역 분할`이기 때문이다. 목록 아이템이나 상세 화면을 미리 넣지 않는다.

### 4. 컴포넌트 분리 방식

권장 구조는 아래와 같다.

```text
HomePage
├─ 모바일 공통 상단 앱바
├─ 모바일 콘텐츠 분기
│  ├─ 기본: MobileAreaBlock(기존 공통 블록)
│  └─ 시세 / 차트 목록: MarketChartMobileListLayout
└─ 모바일 하단 탭
```

`MarketChartMobileListLayout`는 현재는 단일 영역 라벨만 렌더링한다. 이후 상세 화면이 필요해지면 `MarketChartMobileDetailLayout` 같은 별도 컴포넌트를 추가하는 방식으로 확장할 수 있다.

## 예상 구조

```text
모바일 /market-chart
├─ 상단 앱바 영역 - 시세 / 차트
├─ 마켓 목록 영역
└─ 하단 탭 영역
```

```text
모바일 /dashboard, /investment-status, /settings
├─ 상단 앱바 영역 - 메뉴명
├─ 콘텐츠 영역 - 메뉴명
└─ 하단 탭 영역
```

## 검증 기준

아래 조건을 만족하면 이번 레이아웃 작업은 완료로 본다.

- 모바일 `/market-chart`에서 `마켓 목록 영역`이 보인다.
- 모바일 `/market-chart`에서 `콘텐츠 영역 - 시세 / 차트`는 더 이상 보이지 않는다.
- 다른 모바일 메뉴에서는 기존 `콘텐츠 영역 - 메뉴명` 표현이 유지된다.
- 데스크톱 `시세 / 차트` 레이아웃에는 영향이 없다.

## 구현 시 유의사항

- 이번 단계는 목록 화면의 영역 구분만 다룬다.
- 모바일 상세 화면 구현을 앞당겨 넣지 않는다.
- `HomePage.tsx`에서 조건 분기는 하되, 실제 마켓 목록 표현은 전용 컴포넌트로 분리한다.

## 가정

- 모바일에서는 초기 진입 시 마켓이 선택되지 않은 상태가 맞다.
- 상세 화면은 이후 `/market-chart/:market` 라우트로 별도 추가될 예정이다.
- 현재 단계에서는 실제 목록 아이템 대신 하나의 큰 `마켓 목록 영역` 블록으로 충분하다.
