# 시세 차트 마켓 목록 UI 설계서

## 개요

- 작성일: 2026-03-17
- 대상 경로: `frontend/src/features/market-chart/`
- 목적: `시세 / 차트` 탭의 마켓 목록 영역을 업비트에 가까운 고밀도 리스트 패널 형태로 정의해, 이후 백엔드 마켓 조회 API와 자연스럽게 연결할 수 있는 UI 구조를 고정한다.

## 목표

- 데스크톱 `시세 / 차트` 화면의 오른쪽 마켓 목록 패널을 거래소 앱 스타일의 고밀도 리스트로 표현한다.
- 상단에는 `원화 / BTC / USDT` 탭만 두고, 본문은 `종목명 / 현재가 / 전일대비 / 거래대금` 구조로 정렬한다.
- 마켓 이름은 거래소별 데이터 차이를 흡수할 수 있도록 `display_name_ko -> display_name_en -> base_asset` 우선순위로 표시한다.
- 선택된 마켓은 행 전체에 아주 연한 배경을 적용해 현재 선택 상태를 확인할 수 있게 한다.

## 범위

- 데스크톱 `시세 / 차트`의 오른쪽 마켓 목록 패널 UI 구조
- 상단 quote 탭 구성
- 컬럼 헤더 구성
- 리스트 행 구조와 표시 우선순위
- 백엔드 `GET /api/v1/markets` 응답 필드와의 표시 매핑 규칙

## 비범위

- 실제 백엔드 API 호출 연결
- 검색창
- `보유`, `관심` 탭
- 즐겨찾기, 핀, 배지 아이콘 영역
- 실제 정렬 인터랙션
- 모바일 목록 화면 상세 UI
- 다중 거래소 전환 UI

## 설계 결정

### 1. 카드형이 아니라 업비트 근접형 고밀도 리스트로 간다

마켓 목록 패널은 차트 오른쪽의 보조 패널이므로, 카드형보다 정보 밀도가 높은 리스트형이 적합하다.

이번 1차 방향은 아래와 같다.

- 행 높이는 낮고 간격은 촘촘하게 유지한다.
- 숫자 컬럼은 우측 정렬한다.
- 리스트는 패널 내부에서만 독립 스크롤된다.
- 시각적 주영역은 여전히 차트이고, 마켓 목록은 빠르게 훑는 탐색 패널 역할을 맡는다.

### 2. 상단 탭은 `원화 / BTC / USDT`만 둔다

상단 탭은 quote 자산 필터 역할만 먼저 맡는다.

- `원화`
- `BTC`
- `USDT`

`보유`와 `관심`은 이번 범위에서 제외한다. 검색창도 넣지 않는다. 현재 단계는 리스트 밀도와 행 표현을 먼저 맞추는 것이 목적이기 때문이다.

### 3. 컬럼 제목은 `한글명`이 아니라 `종목명`으로 일반화한다

업비트 데이터만 가정하면 한글명이 자연스럽지만, 이후 바이낸스처럼 영문 이름만 있는 거래소가 추가될 수 있다. 따라서 첫 번째 컬럼은 `한글명`이 아니라 `종목명`으로 정의한다.

표시 우선순위는 아래와 같다.

1. `display_name_ko`
2. `display_name_en`
3. `base_asset`

이 규칙이면:

- 업비트는 `비트코인`
- 바이낸스는 `Bitcoin`
- 이름 데이터가 없으면 `BTC`

처럼 자연스럽게 표현할 수 있다.

### 4. 심볼 표기는 `raw_symbol` 원문이 아니라 `base_asset/quote_asset` 형식으로 통일한다

거래소마다 원본 심볼 형식이 다르므로, 보조 줄은 원문 `raw_symbol`을 그대로 쓰지 않는다.

표시 규칙:

- `BTC/KRW`
- `BTC/USDT`

즉, 행의 두 번째 줄은 `base_asset`과 `quote_asset`을 조합한 UI 전용 표기로 통일한다.

### 5. 선택 상태는 행 전체를 아주 연한 배경으로 강조한다

선택 상태는 현재가 셀만 강조하지 않고, 행 전체에 연한 배경을 주는 방식으로 표현한다.

의도:

- 현재 선택된 마켓을 빠르게 찾을 수 있어야 한다.
- 그러나 오른쪽 패널이 차트보다 더 눈에 띄면 안 된다.

따라서 강조는 강한 테두리보다 얇은 톤의 배경 변화가 적합하다.

## 예상 구조

```text
마켓 목록 패널
├─ 상단 quote 탭
│  ├─ 원화
│  ├─ BTC
│  └─ USDT
├─ 컬럼 헤더
│  ├─ 종목명
│  ├─ 현재가
│  ├─ 전일대비
│  └─ 거래대금
└─ 스크롤 리스트
   ├─ 행 1
   ├─ 행 2
   └─ ...
```

각 행 구조:

```text
행
├─ 왼쪽
│  ├─ 종목명
│  └─ 심볼(base/quote)
├─ 현재가
├─ 전일대비
└─ 거래대금
```

## 백엔드 응답 필드 매핑

백엔드 설계 문서 기준 `GET /api/v1/markets` 응답은 아래처럼 매핑한다.

- `display_name_ko`, `display_name_en`, `base_asset`
  - `종목명` 표시값 계산에 사용
- `base_asset`, `quote_asset`
  - `BTC/KRW` 같은 보조 심볼 줄 생성에 사용
- `trade_price`
  - `현재가`
- `signed_change_rate`
  - `전일대비`
- `acc_trade_volume_24h`
  - `거래대금`
- `has_warning`
  - 이후 경고 배지 확장용으로 남겨두되, 이번 1차 UI에는 필수로 넣지 않는다

## 시각적 방향

- 전체 톤은 거래소 앱처럼 밝고 선명한 표 기반 UI에 가깝다.
- 탭은 상단에 납작하게 붙고, 활성 탭만 강조한다.
- 컬럼 헤더는 본문보다 작고 옅은 회색 톤으로 둔다.
- 종목명 첫 줄은 진하고, 두 번째 줄 심볼은 더 작고 옅게 둔다.
- 숫자 컬럼은 모두 오른쪽 정렬한다.
- 등락률과 가격은 향후 데이터 연결 시 상승/하락 색을 적용할 수 있도록 구조를 분리한다.

## 검증 기준

아래 조건을 만족하면 이번 UI 설계는 구현 계획으로 넘어갈 수 있다.

- 오른쪽 마켓 목록 패널이 카드형이 아니라 고밀도 리스트형으로 정의돼 있다.
- 상단 탭이 `원화 / BTC / USDT` 3개만 포함한다.
- 컬럼 제목이 `종목명 / 현재가 / 전일대비 / 거래대금`으로 정리돼 있다.
- 이름 표시는 `display_name_ko -> display_name_en -> base_asset` 우선순위를 따른다.
- 보조 심볼 줄은 `base_asset/quote_asset` 형식으로 통일한다.
- 선택 상태는 행 전체의 연한 배경으로 표현한다.

## 가정

- 데스크톱에서는 기본 마켓이 이미 선택된 상태이므로, 리스트에서도 선택 행이 하나 존재한다.
- 거래소 선택 자체는 현재 왼쪽 메인 컬럼 상단 영역이 맡고, 오른쪽 패널은 quote 탭과 목록 표현에 집중한다.
- 실제 정렬, 무한 스크롤, 1초 재조회는 이후 데이터 연동 단계에서 붙인다.
