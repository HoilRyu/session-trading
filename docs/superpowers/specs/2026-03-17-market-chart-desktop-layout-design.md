# 시세 차트 데스크톱 레이아웃 설계서

## 개요

- 작성일: 2026-03-17
- 대상 경로: `frontend/src/pages/MarketChartPage.tsx`
- 목적: `시세 / 차트` 화면의 데스크톱 콘텐츠 영역을 실제 기능 없이 레이아웃 구역만 보이도록 분할하기

## 목표

- `시세 / 차트` 데스크톱 화면의 정보 구조를 먼저 고정한다.
- 차트가 주영역으로 읽히도록 배치를 잡는다.
- 거래소 선택/요약 정보, 차트, 마켓 목록의 영역 경계를 배경색과 라벨로 확인할 수 있게 만든다.

## 범위

- 데스크톱 `시세 / 차트` 화면의 콘텐츠 영역 레이아웃 분할
- 왼쪽 메인 컬럼과 오른쪽 마켓 목록 패널의 비율 고정
- 왼쪽 상단 정보 패널과 하단 차트 영역 분할
- 각 영역에 확인용 라벨 텍스트 표시

## 비범위

- 모바일 `시세 / 차트` 화면 조정
- 실제 거래소 선택 UI 구현
- 실제 차트 라이브러리 연결
- 마켓 목록 데이터 렌더링
- 상단 정보 패널 내부 세부 카드 구성
- 인터랙션, API 연동, 상태관리

## 설계 결정

### 1. 전체 구조

데스크톱 `시세 / 차트` 콘텐츠 영역은 아래 2열 구조를 사용한다.

- 왼쪽 메인 컬럼: 약 `72%`
- 오른쪽 마켓 목록 패널: 약 `28%`

핵심 원칙은 `차트가 가장 크게 보이는 것`이다. 오른쪽 패널은 보조 정보 영역으로 취급하고, 왼쪽 메인 컬럼이 시각적 중심이 되도록 한다.

### 2. 왼쪽 메인 컬럼 구조

왼쪽 메인 컬럼은 세로 2단 구조를 사용한다.

- 상단: `거래소 선택 + 현재가/요약 정보 영역`
- 하단: `차트 영역`

상단 정보 패널은 메인 컬럼 전체 폭을 가로지른다. 이전 시안처럼 `좁은 왼쪽 컬럼 안에 거래소 선택만 두는 구조`는 사용하지 않는다. 그 구조는 차트보다 마켓 목록이 더 크게 읽혀 의도와 맞지 않기 때문이다.

### 3. 상단 정보 패널 높이

상단 정보 패널은 `압축형`을 채택한다.

- 선택안: `A. 상단 패널 압축형`
- 이유: 차트 세로 높이를 더 많이 확보할 수 있고, 현재 단계의 우선순위인 `차트 중심 레이아웃 확인`에 더 적합하다.

이 단계에서는 상단 패널 내부를 다시 세부 분할하지 않는다. 우선 하나의 큰 영역으로 두고, 이후 필요할 때 내부 구성만 별도 설계한다.

### 4. 오른쪽 마켓 목록 패널

오른쪽 패널은 고정된 보조 패널 역할로 둔다.

- 메인 컬럼보다 존재감이 강해 보이지 않도록 폭을 제한한다.
- 현재 단계에서는 내부 리스트/탭/검색창 구분 없이 하나의 큰 `마켓 목록 영역`으로 표시한다.

## 예상 구조

```text
시세 / 차트 콘텐츠 영역
├─ 왼쪽 메인 컬럼 (약 72%)
│  ├─ 거래소 선택 + 현재가/요약 정보 영역
│  └─ 차트 영역
└─ 오른쪽 마켓 목록 패널 (약 28%)
   └─ 마켓 목록 영역
```

## 검증 기준

아래 조건을 만족하면 이번 레이아웃 작업은 완료로 본다.

- 데스크톱 `시세 / 차트` 화면에서 차트 영역이 가장 큰 시각적 영역으로 보인다.
- 왼쪽 상단 정보 패널이 메인 컬럼 전체 폭을 차지한다.
- 오른쪽 마켓 목록 패널이 보조 영역처럼 읽힌다.
- 각 영역에 텍스트 라벨이 있어 현재 분할 구조를 바로 확인할 수 있다.

## 구현 시 유의사항

- 이번 단계는 레이아웃 확인용 작업이므로 실제 기능은 넣지 않는다.
- 기존 공통 레이아웃 패턴을 크게 깨지 않는 선에서 `MarketChartPage` 전용 콘텐츠 영역만 다르게 구성한다.
- 색상과 라벨은 영역 식별이 목적이며, 최종 디자인으로 간주하지 않는다.

## 가정

- 현재 검토 대상은 `데스크톱 화면`만이다.
- 오른쪽 마켓 목록 패널은 `28%` 전후의 보조 패널 비율로 충분하다.
- 상단 정보 패널 내부 상세 구성은 이후 작업에서 별도로 조정한다.
