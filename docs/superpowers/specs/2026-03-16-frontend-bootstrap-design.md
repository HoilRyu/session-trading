# 프론트엔드 부트스트랩 설계서

## 개요

- 작성일: 2026-03-16
- 대상 경로: `frontend/`
- 목적: `frontend` 폴더에 `pnpm` 기반 `React + Vite + TypeScript + Tailwind CSS + react-router-dom` 실행 환경을 구성하기

## 목표

- 프론트엔드 개발을 시작할 수 있는 최소 실행 환경을 만든다.
- 라우팅이 포함된 기본 앱 구조를 만든다.
- UI 설계나 화면 구현 없이도 로컬 개발 서버와 테스트가 동작하는 상태를 만든다.

## 범위

- `frontend`를 Vite 기반 React TypeScript 프로젝트로 초기화하기
- Tailwind CSS를 Vite 환경에 연결하기
- `react-router-dom` 기반 기본 라우팅 골격 만들기
- 최소 페이지 2개 구성하기
  - 홈 경로 `/`
  - 정의되지 않은 경로 대응용 `*`
- 앱 실행 검증용 최소 테스트 구성하기
- 개발에 필요한 기본 스크립트 구성하기

## 비범위

- 실제 서비스 UI 디자인 작업
- 인증, API 통신, 전역 상태관리, 폼 라이브러리 도입
- 디자인 시스템, 컴포넌트 라이브러리, 아이콘 세트 도입
- 배포 설정, CI 설정, 환경변수 체계 설계
- 백엔드 연동

## 설계 결정

### 1. 초기화 방식

- `pnpm create vite`를 사용해 `React + TypeScript` 템플릿 기반 프로젝트를 생성한다.
- 생성 직후 불필요한 예제 코드는 제거하고, 프로젝트 목적에 맞는 최소 구조로 정리한다.

### 2. 앱 구조

아래처럼 책임이 분리된 최소 구조를 사용한다.

```text
frontend/
  index.html
  package.json
  tsconfig.json
  tsconfig.app.json
  tsconfig.node.json
  vite.config.ts
  src/
    app/
      App.tsx
      App.test.tsx
      router.tsx
    pages/
      HomePage.tsx
      NotFoundPage.tsx
    styles/
      globals.css
    test/
      setup.ts
    main.tsx
```

- `src/app/App.tsx`: 앱 루트 구성만 담당한다.
- `src/app/App.test.tsx`: 앱 루트 렌더링과 기본 라우팅 검증만 담당한다.
- `src/app/router.tsx`: 라우트 정의만 담당한다.
- `src/pages/`: URL 단위 페이지 파일만 둔다.
- `src/styles/globals.css`: Tailwind CSS와 전역 스타일 진입점만 둔다.
- `src/test/setup.ts`: 테스트 환경 초기화만 담당한다.

### 3. 라우팅 방식

- `react-router-dom`의 브라우저 라우터 기반 구성을 사용한다.
- 이번 단계에서는 중첩 라우트나 레이아웃 라우트는 도입하지 않는다.
- 최소 경로만 정의한다.
  - `/`: 홈 페이지
  - `*`: 잘못된 경로 대응 페이지

이 구성은 현재 요청 범위인 "라우팅이 되는 기본 골격"을 만족하면서도, 이후 페이지가 늘어날 때 `router.tsx`만 확장하면 되도록 한다.

### 4. 스타일링 방식

- Tailwind CSS는 동작 가능한 상태로만 연결한다.
- 전역 스타일 파일에서 Tailwind를 불러오고, 앱 루트에서 해당 파일을 진입시킨다.
- 디자인 토큰, 커스텀 테마, 다크 모드 전략은 이번 단계에서 정의하지 않는다.

### 5. 테스트 방식

- `Vitest`와 `Testing Library`를 사용해 최소 렌더링 테스트를 추가한다.
- 테스트 환경은 브라우저 유사 환경인 `jsdom`을 사용한다.
- 테스트 범위는 다음으로 제한한다.
  - 기본 라우트(`/`) 렌더링 확인
  - 라우터 구성이 깨지지 않았는지 확인

이 테스트는 기능 검증보다는 환경 구성 검증에 초점을 둔다.

## 데이터 흐름

초기 부트스트랩 단계의 흐름은 단순하다.

1. 브라우저가 `src/main.tsx`를 실행한다.
2. `main.tsx`가 전역 스타일과 앱 루트를 로드한다.
3. `App.tsx`가 라우터 프로바이더를 마운트한다.
4. 현재 URL에 따라 `router.tsx`가 해당 페이지 컴포넌트를 렌더링한다.

데이터 저장소나 API 요청은 이번 단계에 포함하지 않는다.

## 오류 처리

- 잘못된 URL 접근은 `*` 라우트에서 처리한다.
- 개발 환경 오류는 Vite 기본 오류 출력에 맡긴다.
- 앱 내부 예외 경계나 서버 오류 화면은 이번 범위에 포함하지 않는다.

## 검증 기준

아래 조건을 만족하면 이번 작업은 완료로 본다.

- `pnpm install` 후 의존성 설치가 완료된다.
- `pnpm dev`로 Vite 개발 서버가 실행된다.
- 기본 홈 경로가 렌더링된다.
- 정의되지 않은 경로에서 `NotFoundPage`가 렌더링된다.
- `pnpm test --run`이 통과한다.

## 구현 시 유의사항

- 예제 카운터 UI와 기본 로고 파일은 제거한다.
- 아직 필요하지 않은 추상화는 추가하지 않는다.
- 사용자 요청 없이 상태관리, 데이터 패칭, UI 라이브러리를 선행 도입하지 않는다.
- 변경은 `frontend` 범위에 한정하고, 백엔드나 데이터베이스 폴더는 건드리지 않는다.

## 가정

- 패키지 매니저는 `pnpm`을 사용한다.
- 현재 단계의 목적은 "UI 구현"이 아니라 "프론트엔드 실행 환경 구성"이다.
- 라우팅은 포함하지만, 실제 화면 설계는 이후 별도 작업으로 진행한다.
